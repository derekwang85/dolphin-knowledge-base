
## Plan B: Media Server for aITMS Image Attachments

**Context**: 之前 aITMS 收不到企微图片消息，因为在 ingress 信封中只传了过期的 PicUrl。
**Solution**: 方案 B — 起一个静态文件服务器，把本地媒体文件暴露为 HTTP 可访问 URL。

### What was done

1. **media-server.py** — Python HTTP server on `:18888`, serves `/home/cbnb/.openclaw/media/`
   - URL: `http://10.88.98.20:18888/inbound/<filename>`
   - Auth: built-in Bearer token support (disabled for dev), `MEDIA_SERVER_TOKEN`
   - Source: `scripts/media-server.py`

2. **aitms_ingress_push.py** — Added media attachment resolver
   - `resolve_media_attachments()` — converts `file://` paths to HTTP URLs
   - Updates `content` field: replaces `[图片] <expired_PicUrl>` with accessible URL
   - `AITMS_MEDIA_BASE_URL` env var (default: `http://10.88.98.20:18888`)
   - Envelope now supports optional `attachments` array

3. **Persistence**:
   - `@reboot` cron fallback: starts server on boot
   - `scripts/aitms-media-server.service` — systemd ready for `sudo` install
   - `scripts/manage-media-server.sh` — management script

### Verifications
- ✅ Media server: localhost:18888 and LAN 10.88.98.20:18888 both return 200
- ✅ Envelope resolver: `[图片] <PicUrl>` → `[图片] http://10.88.98.20:18888/inbound/xxx.png`
- ✅ Push script: routes to aITMS (auth failure expected — separate issue)
