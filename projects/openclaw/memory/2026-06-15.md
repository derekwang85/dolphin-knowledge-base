## 🔧 全天 — WBS回写Bad 90修复

### 根因链 (重要!)
1. **核心bug**: `wbs-mark-done.sh` 的Python heredoc使用`$(python3 -c "...")`双引号包裹，内部`f"..."`和`"🔲"`导致bash引号提前截断 → emoji变裸字符 → SyntaxError静默失败 → 脚本永远返回NO → WBS状态从未更新
2. **衍生bug1**: `|||`格式列偏移 — 部分行前两列为空使整表右移1列，状态列从cells[7]移到cells[8]，v2脚本仍读cells[7]（实际是Agent列），误判"已是✅"跳过更新
3. **衍生bug2**: WBS文件曾因`wbs-cli.py import`截断（15521→532行），37%任务丢失。根因：`import_from_markdown()`解析器只成功导入530/847项，后`_sync_markdown()`用残缺数据覆盖了原文件
4. **衍生bug3**: Refill脚本的scan range只扫到cells[8]，`|||`格式下✅在cells[9+]被漏检

### 修复
- **v2**: 临时文件传参避开bash引号污染
- **v3**: `wbs-cli.py`加99%安全阀（导入<99%拒绝写入）+ 1%偏差校验 (JSON vs MD)
- **最终**: `wbs-mark-done.sh` v3改为调`wbs-cli.py done` → 写JSON → 自动同步MD，不再直接编辑.md
- refill scan range从range(4,9)扩展到range(4,13)
