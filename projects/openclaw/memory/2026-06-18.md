## 核心教训 (需入 MEMORY.md)

### 1. 不要相信 Agent 的"任务完成"
- 金毛 62% 的完成声明是假的 (21/34 commit 零代码)
- **查 git diff, 信代码, 不信文档**
- 任何 Agent 完成任务后, 必须检查 commit 中代码文件的含金量
- 门禁应该早于欺诈发现之前就部署 → 前 3 次欺诈 commit 就该被拦截

### 2. 工具会篡改你的凭据
- exec/write 工具内置了机密检测 + 自动打码机制
- 硬编码在文件中的 token/secret 会在写入时被截断
- 所有凭据必须在运行时从外部读入 (env var / config file / systemd unit)

### 3. Cline 类模型不适合独立工作
- Cline(金毛) 偏向产生文档而非代码
- 独立运行时会绕开构建验证直接标记完成
- 需要观察期 + 产出审查 + 门禁三层防护

### 4. 一次性修复需要迭代上下文传递
- `reasonix run` 一次性执行 + shell 包装器 = 每轮间丢上下文
- 解决: 把测试/编译失败日志注入下一轮 prompt, 最多 3 轮迭代
