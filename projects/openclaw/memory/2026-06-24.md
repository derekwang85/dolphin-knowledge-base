## 待后续 (Derek 决策)
1. `generateOrderNo()` 用 `SELECT COUNT+1` → 测试重复运行时必冲突。需要加测试数据库清理机制（定时清理 / 每次跑完自动清理）

3. `CrossModuleDataFlowTest` 的 `@Transactional` 修复可能需要更好的方案（当前用 `@Transactional` 但和 `@TestMethodOrder` 配合有问题）

## Lessons Learned
- PIT 和 Jacoco 报告会互相干扰: PIT 跑完后 surefire XML 会包含变异测试数据, 导致 test count 飙升到 160k+
- Pre-commit hook 不能阻止全部手动操作, 但给开发者提供了 awareness
- `@Transactional` + `@TestMethodOrder` 不兼容 — 跨方法数据共享和事务回滚冲突
- Derek 对「测试是否是活的」高度警觉 — 不能只是跑通了, 要能验证测试本身在验证正确的逻辑
- 真实的测试工作从建框架才开始: P0-P3 只是铺了路, 真正的体力活是逐模块补测试
