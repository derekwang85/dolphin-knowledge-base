## 事件：08:18 用户报页面渲染异常

用户反馈 admin/ops01 登录后所有admin菜单报页面渲染异常和500错误。

## 诊断过程

### 远程分析 → E2E 验证 → 根因定位

1. 最初推断为远程服务器（10.8.211.161:3000）Vite Dev Server 502 问题
2. Derek 纠正：本地（10.88.97.77:5173）TradeOMS 前端也有同样问题
3. E2E 复现确认：
   - OA 账单: `GET /api/v1/oa/invoice` → 500
   - VA 虚拟账户: `GET /api/v1/va/balance` 等 → 400 "companyId is required"
4. 核心根因：前端 store/*-api.js 里的 API 路径和后端 @RequestMapping 实际注册的路径对不上

### 关键教训
- 测试体系缺少**前后端集成契约校验**维度
- 前端 store/*-api.js 各模块有自己的 baseURL（`currency-api.js` 用 `/api/v1/master-data/currencies` 等），但之前没人交叉验证
- 复数/单数路径不匹配是重复模式（ISS-089 OA invoice + ISS-098 OA credit）

## 修复清单

| 问题 | 根因 | 修复 |
|------|------|------|
| OA 账单500 | 前端调 `/invoice`(单数)，后端只有 `/invoices`(复数) | OaController 加 `@GetMapping({"/invoices", "/invoice"})` |
