# ADR-001: 双轨并行架构设计

> **状态**: 已接受 | **日期**: 2026-06-09 | **作者**: aITMS01

---

## 背景

OMS 面临两类用户需求冲突：
- **传统用户**：习惯 Web 表单操作，信任完整的操作界面，对 AI 有戒心
- **新潮用户**：希望更快更轻量的交互方式，愿意尝试 AI 辅助

采用单一方案无法同时满足。

## 决策

**选择双轨并行架构**：Java Web（Track A）+ OpenClaw（Track B），共享同一套后端。

```
用户 → 选择入口
  ├── Track A: Web 管理台（React/Vue）→ Java API → MySQL/Qdrant/MinIO
  └── Track B: OpenClaw（WeCom 对话）→ Java API → MySQL/Qdrant/MinIO
```

## 关键约束

1. **后端是唯一业务真源**，两套前端不做任何业务逻辑
2. **同一套 API**，Web 和 OpenClaw 调用完全相同的接口
3. **数据完全共享**，Web 创建的报价在 OpenClaw 可查，反之亦然
4. **Track B 功能是 Track A 的子集**，核心/复杂操作（如 LC 编辑）Web 独占

## 备选方案及舍弃原因

| 方案 | 舍弃原因 |
|------|---------|
| 仅 Web | 无法满足希望快速交互的用户，错失 LLM 增强机会 |
| 仅 OpenClaw | 老派用户无法接受，LC 编辑等复杂操作体验差 |
| 两套独立后端 | 数据不一致，开发量翻倍 |

## 影响

- **开发量**：前端 x2，但后端同一套，节省约 30%
- **维护**：后端一次变更，两前端同时受益
- **用户体验**：用户自由选择，无强制切换成本
- **测试**：核心 API 测试一次，前端测试独立
